# Workflow: Crear Proyecto PBL

## Objetivo
Diseñar un proyecto PBL multi-clase con driving question, milestones y rúbricas.

## Prerequisitos
- Plan mínimo aprobado para el curso
- Al menos 2 temas disponibles para cubrir

## Pasos

### Paso 1 — Análisis de alcance
1. Leer `plan-minimo.md` del curso activo
2. Identificar temas candidatos para el proyecto
3. Estimar duración (2-16 semanas)

### Paso 2 — Driving question
1. Proponer 2-3 driving questions motivadoras
2. Esperar aprobación del docente (GATE OBLIGATORIO)

### Paso 3 — Diseño de milestones
1. Crear 3+ milestones con deliverables específicos
2. Asignar peso porcentual a cada milestone
3. Vincular prerequisite_topics verificados contra plan mínimo
4. Generar rúbricas por milestone

### Paso 4 — Anti-delegación
1. Incluir ≥2 medidas anti-delegación (Denny et al. 2024):
   - Presentación oral obligatoria
   - Peer review cruzado
   - Code walkthrough grabado
   - Diario de aprendizaje individual

### Paso 5 — Integración GitHub (opcional)
1. Si `classroom_enabled: true`, crear repo template grupal
2. Configurar branch por milestone si aplica

### Paso 6 — Output
1. Generar `pbl-{nombre}.json` validado contra schema
2. Generar `pbl-{nombre}.md` legible
3. Generar rúbricas separadas por milestone
